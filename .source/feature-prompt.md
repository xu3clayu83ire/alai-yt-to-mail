# 功能資產：前端 UI 通用改版規範（可重複套用）

> 目的：本文件作為通用前端 UI 開發規範，避免綁定特定專案、品牌、部署平台與商業情境。

---

## 1. 使用原則

1. 本規範僅定義「介面與互動」，不預設任何後端技術、雲端服務、框架或部署流程。
2. 本規範不得包含特定產品名稱、公司名稱、專案代號、網址、環境名稱或個人情境描述。
3. 若功能已有商業邏輯，UI 改版時應維持既有邏輯行為一致，除非需求明確要求調整。
4. 文件內的數值為建議基準，可依產品定位微調，但應維持一致性。

---

## 2. 視覺方向

### 2.1 基礎設計語彙

| Token | 建議值 |
|:------|:------|
| 頁面背景 | `#fafafa` |
| 區塊背景 | `#ffffff` |
| 區塊邊框 | `1px solid #efefef` |
| 圓角 | `16px - 24px` |
| 陰影 | `0 4px 40px rgba(0,0,0,0.06)` |
| 主色漸層 | `linear-gradient(135deg, #f97316, #ec4899)` |
| 文字字體 | `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif` |
| 動畫時間 | `0.2s ease` |

### 2.2 版面原則

1. 以留白與層次感為主，避免裝飾性過高的視覺元素。
2. 主要內容區塊採卡片式結構，使用輕邊框與柔和陰影區分層級。
3. 互動元件狀態需明確區分：預設、hover、active、disabled、error、success。

---

## 3. 元件規範

### 3.1 首屏與資訊區

1. 主標題建議使用 `clamp(28px, 5vw, 48px)`，強調視覺焦點。
2. 次要說明文字建議 `16px - 17px`，行高至少 `1.6`，提高可讀性。
3. 標籤或膠囊元件使用高圓角（如 `999px`）並維持低對比背景。

### 3.2 卡片列表

1. 網格建議：`grid-template-columns: repeat(auto-fit, minmax(260px, 1fr))`。
2. 卡片 hover 動效建議包含位移與陰影，不超過 `translateY(-2px)`。
3. 卡片資訊層級：圖示 → 標題 → 說明，字級由大到小清楚遞減。

### 3.3 表單與上傳互動

1. 拖放區建議使用虛線邊框，並提供 idle、selected、disabled 三種狀態。
2. 當流程進行中，需停用互動並提供視覺提示（如 `opacity` 與 `cursor`）。
3. 檔案名稱過長時需處理溢出（`word-break` 或 `ellipsis`）。

### 3.4 進度與狀態

1. 進度條需同時呈現視覺進度與數值百分比。
2. 步驟狀態至少提供：待處理、進行中、完成、錯誤。
3. 進行中動畫需低干擾，轉速建議 `0.8s - 1.2s` 線性循環。

### 3.5 錯誤提示

1. 錯誤區塊需以背景色與邊框雙重提示，不僅依賴文字敘述。
2. 錯誤文案應具體指出可執行的修正方向。

---

## 4. 無障礙與易用性

1. 表單輸入需保留可聚焦與可鍵盤操作能力。
2. 可點擊區域需有可辨識的焦點樣式。
3. 文字與背景對比應符合可讀性要求，避免過淡灰階組合。
4. 圖示不可單獨承載關鍵語意，需搭配文字說明。

---

## 5. 響應式規範

1. 手機寬度下，卡片與按鈕應維持單欄閱讀優先。
2. 膠囊標籤與按鈕在窄版需可換行或自動縮排，不可溢出容器。
3. 主流程操作按鈕在手機需維持可點擊高度（建議至少 `44px`）。

---

## 6. 驗收清單

- [ ] 所有核心元件皆具備完整狀態（default / hover / active / disabled / error / success）
- [ ] 主要流程在桌機與手機皆可順利完成
- [ ] 長文字與極端內容不會破版
- [ ] 互動元件具備鍵盤可操作性
- [ ] 錯誤訊息具可行動性且位置明確
- [ ] 動效不影響操作節奏，且無過度閃爍
- [ ] 樣式規則可跨專案重用，不含任何特定專案識別資訊
